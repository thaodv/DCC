1. IPFS简介

IPFS用基于内容的寻址替代传统的基于域名的寻址，用户不需要关心服务器的位置，不用考虑文件存储的名字和路径。我们将一个文件放到IPFS节点中，将会得到基于其内容计算出的唯一加密哈希值。哈希值直接反映文件的内容，哪怕只修改1比特，哈希值也会完全不同。当IPFS被请求一个文件哈希时，它会使用一个分布式哈希表找到文件所在的节点，取回文件并验证文件数据。

IPFS是通用目的的基础架构，基本没有存储上的限制。大文件会被切分成小的分块，下载的时候可以从多个服务器同时获取。IPFS的网络是不固定的、细粒度的、分布式的网络，可以很好的适应内容分发网络的要求。

2.下载IPFS

通过官网地址 https://dist.ipfs.io/#go-ipfs 可以下载最新版本压缩包，官方提供多平台版本。

注：访问IPFS官方网站需要科学上网。

3.安装IPFS

下载后，解压缩归档文件，然后使用install.sh脚本将ipfs二进制文件移动到可执行文件$ PATH中的某个位置：

tar xfz go-ipfs_v0.4.13_linux-amd64.tar.gz

cd go-ipfs

./install.sh

IPFS就装好了，运行ipfs help检验是否可以正确运行

# ipfs help

初始化一个工作区

# ipfs init

添加文件到IPFS

# echo "Hello world" > my.txt

# ipfs add my.txt

added QmePw8gVcBMb8x6kAep6aMBAX23hCSk6iZW3i9VKkiFhu1 my.txt

修改配置

#export EDITOR=vim

#ipfs config edit

得到的内容中，将

"API": "/ip4/127.0.0.1/tcp/5001"修改为

"API": "/ip4/0.0.0.0/tcp/5001"，然后 :wq退出

启动Daemon服务

# ipfs daemon

启动之后，就可以通过HTTP访问文件内容，比如刚才的文件

# ipfs cat QmePw8gVcBMb8x6kAep6aMBAX23hCSk6iZW3i9VKkiFhu1

Hello world

4.使用自定义节点

创建自己的节点后，在BitExpress APP上可以使用自己的节点作为自定义节点。但为了保证您的节点与BitExpress官方节点的之间的数据同步速度，还需要把官方的两个节点和自己的节点都加入到网络

官方节点

/ip4/47.100.242.139/tcp/4001/ipfs/QmVHMddQWm18m485N6caFfYrzDnDyLRvQaVvubiAPHbeUY

/ip4/47.100.238.10/tcp/4001/ipfs/QmQhAn5FmJ8cWVffzLAowE8ao4K5BXeakaZ54R4P9esaKV

查看自己的节点

# ipfs id

添加和删除

#ipfs bootstrap add 上面展示的节点 添加启动节点

added /ip4/47.100.238.10/tcp/4001/ipfs/QmQhAn5FmJ8cWVffzLAowE8ao4K5BXeakaZ54R4P9esaKV

ipfs bootstrap rm 删除启动节点

查看自己的节点

# ipfs bootstrap list

可以看到刚才添加的节点
